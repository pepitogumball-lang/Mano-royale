# Contribuyendo a Mano Royale\n\nGracias por tu interés en mejorar Mano Royale. Este documento te guiará sobre cómo contribuir.\n\n## Cómo Reportar Bugs\n\n### Antes de reportar\n1. Verifica que uses la última versión del add-on\n2. Comprueba tus Experimental Features están habilitadas\n3. Lee el TECHNICAL.md para troubleshooting conocido\n\n### Información a incluir\n- Versión de Minecraft Bedrock\n- Sistema operativo\n- Pasos para reproducir\n- Comportamiento esperado vs actual\n- Logs si están disponibles\n\n## Cómo Sugerir Mejoras\n\nLas sugerencias son bienvenidas. Por favor incluye:\n- Descripción clara de la mejora\n- Por qué crees que es útil\n- Ejemplos si es posible\n\n## Proceso de Desarrollo\n\n### Configurar Entorno de Desarrollo\n\n1. Clona el repositorio:\n```bash\ngit clone https://github.com/pepitogumball-lang/Mano-royale.git\ncd Mano-royale\n```\n\n2. Ubica tus packs en el directorio de desarrollo de Minecraft:\n   - Windows: `%APPDATA%\\.minecraft\\development_packs\\`\n   - macOS/Linux: `~/.minecraft/development_packs/`\n\n3. Abre Minecraft y carga el add-on\n\n### Editando el Código\n\n**Para Scripts (JavaScript)**:\n- Usa VSCode con extensión de Minecraft\n- Sigue el formato existente\n- Añade comentarios para código complejo\n- Prueba cambios en juego\n\n**Para Funciones (mcfunction)**:\n- Crea nuevas funciones en `behavior_pack/functions/`\n- Usa convención de nombres: `descriptivo.mcfunction`\n- Test con `/function manoroyale:nombre_funcion`\n\n**Para Items (JSON)**:\n- Mantén formato consistente\n- Valida JSON antes de commitear\n- Prueba texturas y propiedades en juego\n\n**Para UI (JSON)**:\n- Sigue estructura de hud_screen.json\n- Prueba en diferentes dispositivos\n- Verifica offsets y tamaños\n\n## Estándares de Código\n\n### JavaScript\n```javascript\n// ✅ CORRECTO\n/**\n * Descripción clara de la función\n * @param {Type} nombre - Descripción del parámetro\n * @returns {Type} Descripción del retorno\n */\nfunction nombreFuncion(parametro) {\n    try {\n        // Lógica aquí\n        return resultado;\n    } catch (error) {\n        console.error(`[Mano Royale] Error: ${error.message}`);\n        return null;\n    }\n}\n\n// ❌ INCORRECTO\nfunction f(x){return x}\n```\n\n### JSON\n```json\n{\n  \"format_version\": \"1.20.50\",\n  \"minecraft:item\": {\n    \"description\": {\n      \"identifier\": \"namespace:identifier\"\n    },\n    \"components\": {}\n  }\n}\n```\n\n## Enviar Cambios\n\n1. Crea una rama para tu feature:\n```bash\ngit checkout -b feature/nombre-feature\n```\n\n2. Haz commits descriptivos:\n```bash\ngit commit -m \"[tipo] Descripción clara del cambio\"\n# Tipos: [fix], [feat], [docs], [refactor], [test]\n```\n\n3. Push a tu rama:\n```bash\ngit push origin feature/nombre-feature\n```\n\n4. Abre un Pull Request con:\n   - Descripción clara de cambios\n   - Por qué son necesarios\n   - Cualquier testing realizado\n\n## Estructura de Commits\n\n```\n[tipo] Descripción breve (50 chars max)\n\nDescripción más detallada si es necesario.\nExplica el contexto y razón del cambio.\n\nFixes #123\n```\n\nTipos de commits:\n- `[fix]` - Corrección de bugs\n- `[feat]` - Nueva funcionalidad\n- `[docs]` - Cambios de documentación\n- `[refactor]` - Refactorización de código\n- `[test]` - Agregando o actualizando tests\n- `[chore]` - Tareas de mantenimiento\n\n## Proceso de Review\n\n1. Un mantenedor revisará tu PR\n2. Se pueden solicitar cambios\n3. Una vez aprobado, se mergeará a main\n4. Tu contribución aparecerá en el próximo release\n\n## Convenciones de Nombres\n\n### Archivos\n- `camelCase.js` para scripts\n- `snake_case.mcfunction` para funciones\n- `snake_case.json` para configuración\n\n### Variables (JavaScript)\n```javascript\nconst CONSTANT_NAME = value;      // Constantes\nlet variableName = value;          // Variables\nfunction functionName() {}         // Funciones\nclass ClassName {}                 // Clases\n```\n\n## Licencia\n\nAl contribuir, aceptas que tu código sea distribuido bajo la misma licencia del proyecto.\n\n## Preguntas?\n\nSi tienes preguntas sobre cómo contribuir, abre una issue o contacta a los mantenedores.\n\n---\n\n**¡Gracias por contribuir a Mano Royale!** 🎮\n"
